사람의 기억
: 텅 빈 머리에 기억이 채워진다.
: 일상이 평소와 대부분 비슷하여 연속성이 있는 삶이 이뤄진다.
: 순서대로 떠오르는 것도 아니구, 믿을 만한 것도 아니다.

글쓰기
: 사람의 부족 기억력을 채워주기 위해 개발
: 자신의 멋진 아이디어를 읽고 각본을 작성한다.
: 어떤 것을 저장하고 나중에 접근해서 살펴보는 용도

메모리
: 두 작업의 시점을 이어주기 위해서 정보를 보관해 두는 것
: 종이는 글로 이뤄진것, 영화는 자기테이프
: 텔레그랩에 사용되었던 릴레이도 "논리게이트"형태의 "플립플롭"으로
  1바이트를 저장할 수 있다.

쓰기 신호 
: 데이터 입력이 회로에 저장될 수 있도록 만드는 것
: 0의 값을 가지고 있어 입력의 변화가 출력에 대해여 영향을 끼치지 못한 것

쓰기 입력[쓰기 신호 저장]
: 1로 바꾸었다가 다시 0으로 변경하면된다.
: 데이터 값의 변화를 막을 수 있기 때문이다.


8개의 1비트 래치
: 한 신호가 서로 다른 8개의 어떤 값을 가졌는지 저장하고 싶을 경우
// 8개의 1비트 값들을 하나씩 래치에 저장하고 싶은 것
: 8개의 입력으로부터 선택을 하려고 한다면, 3개의 스위치가 필요하다.
// 무엇은 8개의 입력, 출려과연결은 3개의 입력

출력부분
: 입력 값을 지정하기 위해 8:1선택기가 요기에서는 필요할 것이다.
  // 8개의 데이터 입력, 3개의 선택 입력

입력부분
: 데이터 입력 신호 , 쓰기 입력 신호
-> 데이터 입력을 한꺼번에 연결가능
-> 한 비트 쓰기 신호가 오직 하나의 래치에만 전달될 수 있도록 만들어야 함.
3-8디코더 : 8:1선택기와 반대 동작을 수행할 수 있는 회로

주소
: 세 비트의 선택 신호가 디코더와 선택기에 같이 들어가고 있음
: 어떤 레치에 접근할 것인지 알려주는 역할
: 데이터 입력 값이 저장되도록 쓰기 신호를 변경시킬지 판단하기 위하여
  입력으로 사용된다.
: RAM이란 이름으로 임의 접근 메모리로 1비트 값 8개를 저장할 수 있음

메모리
: 정보를 보전할 수 있는 메모리
: 어떤 값이 저장되었는 지 확인이 가능하여 읽고/쓸 수 있어야 한다.
: 몇몇 메모리는 값을 순차적을 읽어야 하는 것 "RAM 배열"이라고 부른다.

8 * 1 형태 [RAM 배열]
: 랩의 배열로 구성된 것
: 두 값을 곱한 결과는 이 메모리 배열에 저장될 수 있는 비트 수
: 8 * 2 RAM의 배열 -> [8 * 1] + [8 * 1] 
  // 8개의 값을 저장하고 2비트 크기를 가진다.
: 16 * 1 RAM의 배열 -> 1비트 값 16개를 저장할 수 있음.RAM


RAM 배열
: 저장할 수있는 값의 수 -> 주소 입력의 비트 수와 직접적인 연관을 가짐
: 주소 입력이 없다면, 하나의 값만이 저장될 수 있음.
: 주소 입력이 한 비트라면 2개의 값을 저장하는 것이 가능하다.
:  주소 입력이 두 비트라면 4개의 값을 저장하는 것이 가능하다.
// RAM 배열에 저장되는 값의 수 = 2 ^ 주소 입력의 비트 수
-> 근사 등가 : 어떤 수와 2의 거즙 제공수의 근사값

[취득 스위치]
메모리의 제어권 : 스위치의 1인 경우 제어찬은 메모리에 대한 제어권 획득
메모리 전송 속도의 단위 : 통신 회선을 통하여 전달될 때의 전송 속도

RAM[임의 접근 메모리] -> volatile memory[휘발성 메모리]
